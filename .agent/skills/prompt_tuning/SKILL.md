# Prompt Tuning Skill (プロンプト改善スキル)

## 概要
`settings/prompts/` 以下のLLMプロンプトを改善するためのスキルです。
Issue #194のような「プロンプト漏れ」や「期待と異なる出力形式」の問題を解決する際に活用します。

---

## 基本方針

1. **観察ファースト**: 修正前に必ずLLMの生の応答をログで確認する
2. **二重防御**: プロンプト修正とパーサー強化の両方を検討する
3. **増分改善**: 一度に大きく変えず、小さな変更で効果を検証する
4. **ネガティブ制約**: 「〜してはならない」という明確な禁止事項を含める

---

## 🔍 問題分析フェーズ

### 1. LLM出力の取得
```python
# tribute_generator.py 等の呼び出し元でログを追加
logger.info(f"=== LLM Response [{prompt_type}] ===")
logger.info(raw_response)
logger.info(f"=== End LLM Response ===")
```

### 2. 問題パターンの分類

| パターン | 症状 | 原因 | 対策 |
|---------|------|------|------|
| **中間出力漏れ** | 「ステップ1」等が表示される | プロンプトの思考手順を出力 | 「最終出力のみ」を明記 |
| **会話調混入** | 「了解しました」等が含まれる | 会話モードの応答 | 「説明文不要」を明記 |
| **重複エントリ** | 同じ内容が複数回出現 | 段階的出力を全て返す | パーサーで重複排除 |
| **プレフィックス** | `: 過去に...`等で始まる | フォーマット未遵守 | 正規表現でクリーンアップ |
| **コードブロック** | ``` で囲まれる | Markdown装飾の過剰 | 「コードブロック不要」を明記 |

## ✏️ プロンプト修正フェーズ

### 推奨する制約の追加例

```markdown
## 出力ルール（厳守）

> [!CAUTION]
> 以下を必ず守ること

1. **思考プロセスを出力しない**: 「ステップ1」「まず〜」等の中間出力は一切含めない
2. **会話調は不要**: 「了解しました」「以下が〜です」等の前置きは含めない
3. **最終フォーマットのみ**: 指定された出力形式のみを出力する
4. **コードブロック不要**: ``` で囲まない
5. **該当なしは空文字**: 該当がない場合は何も出力しない（空文字）
```

### プロンプト構造のテンプレート

```markdown
# 調査目的
[背景と目的を簡潔に]

# 入力データ
[変数やコンテキスト]

# 調査手順（内部処理用、出力には含めない）
1. [ステップ1]
2. [ステップ2]

# 出力ルール
[上記の制約]

# 出力フォーマット
[厳密なフォーマット指定]

# 出力後のセルフチェック
[LLMに自己検証を促す]
```

## 🛡️ パーサー強化フェーズ

プロンプト修正だけでは完全に制御できないため、パーサー側でも防御します。

### 共通クリーンアップ処理

```python
import re

def clean_llm_output(text: str) -> str:
    """LLM出力の共通クリーンアップ"""
    if not text:
        return ""
    
    # コードブロックの除去
    text = re.sub(r"```[a-z]*\n?", "", text)
    text = re.sub(r"```", "", text)
    
    # 中間出力キーワードを含む行を除去
    skip_patterns = [
        r"^ステップ\d+[：:].*$",
        r"^了解.{0,20}$",
        r"^以下.{0,30}です[。.]?$",
        r"^最終出力[：:]?$",
    ]
    lines = text.split("\n")
    filtered = [l for l in lines if not any(re.match(p, l.strip()) for p in skip_patterns)]
    
    return "\n".join(filtered).strip()
```
