# 選手プロフィール本文調査スキル

## 概要

この skill は、選手プロフィールの**本文づくり**に特化します。

担当範囲は以下です。

1. 情報収集
2. `labelled_lines_v1` 形式への整形
3. セルフレビュー
4. `temp/player_profiles_YYYYMMDD_[team]_[player].md` 出力

GCS upload、standalone HTML 生成、deploy は親 skill 側で扱います。

## 情報源の優先順位

| 優先度 | ソース | 用途 |
|--------|--------|------|
| 1 | Wikipedia（英語版）/ Transfermarkt | 基本情報、経歴、移籍金 |
| 2 | クラブ公式、監督会見、選手インタビュー | 発言、移籍の背景、クラブ文脈 |
| 3 | UEFA / Premier League などの公式大会サイト | 受賞、記録、代表実績 |
| 4 | 一般紙・大手メディア | 補助的な裏話、補強報道 |

基本方針:
- まず一次情報と公式情報で骨格を作る
- エピソードは「熱心なファンでもへぇーとなる」ものを 1〜2 個入れる
- 推測は書かない

## `labelled_lines_v1` の書き方

各行は `ラベル::本文` の形式にする。

よく使うラベル:
- `生まれ`
- `国籍`
- `ポジション`
- `身長・利き足`
- `経歴`
- `移籍金`
- `特徴`
- `面白いエピソード`
- `監督・選手の発言`
- `受賞`
- `記録`

## 本文作成ルール

- 最終出力は日本語にする
- `経歴` は 1 行に詰め込まず、必要なら複数行に分ける
- ただし `経歴::` を細かく分けすぎず、読者が流れを追える粒度に圧縮する
- 生い立ち、ブレイクしたクラブ、直前所属、現所属が追える構成を優先する
- ユースとトップ昇格が連続する場合は 1 行にまとめてよい
- ポジションは本職を先に書く
- 監督コメントは、時期や文脈が分かるように短く補足する
- `受賞` と `記録` は意味に応じて使い分ける

## セルフレビュー

最低限、以下を確認する。

1. 経歴が多すぎて流れが見えなくなっていないか
2. ポジション欄が本職よりコンバート先を強く見せていないか
3. 監督・選手の発言欄で別時期のコメントを雑に混ぜていないか
4. ラベル名と本文内容がずれていないか
5. 他選手と比べて 1 人だけ冗長すぎないか

## 出力形式

調査結果は `temp/player_profiles_YYYYMMDD_[team]_[player].md` にまとめる。

推奨構成:

````markdown
# Player Name

```text
生まれ::...
国籍::...
...
```

## Sources

- https://...
- https://...
````

## 良い出力例

```text
生まれ::2003年8月17日、フランス・リヨン
国籍::フランス（アルジェリア系）
ポジション::左ウィンガー / アタッキングMF
身長・利き足::176cm / 左利き
経歴::2010-2024: オリンピック・リヨン（アカデミー→トップチーム）
経歴::2025-: マンチェスター・シティ
移籍金::約4000万ユーロ（報道）
特徴::狭い局面でのターン、右ハーフスペースからのドリブル前進、ラストパスに強み。
面白いエピソード::2025年12月vsサンダーランド戦でラボーナアシストを記録。試合後、グアルディオラ監督は「メッシがそんなクロスを蹴るのを見たことがない。メッシの強みはシンプルさだ」と言及し、婉曲にシンプルさを求めた。
移籍の裏話::複数ビッグクラブが獲得を打診したが、グアルディオラとの直接対話でシティ行きを決断。
```
