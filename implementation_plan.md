# Issue #215 実装計画（カレンダーを主導線に変更）

作成日: 2026-02-23  
対象Issue: [#215](https://github.com/igrekplus/football-delay-watching/issues/215)

## 1. 背景と課題

Issue #215 では、現状の導線が以下のように分散している点が課題です。

- ログイン画面 → レポート一覧 → レポート
- ログイン画面 → レポート一覧 → カレンダー → レポート
- ログイン画面 → カレンダー → レポート

要求される導線は次の2つです。

- ログイン画面 → カレンダー → レポート
- ログイン画面 → カレンダー → レポート一覧 → レポート

ユーザー合意により、導線はシンプルに「カレンダーからレポート一覧へ行くリンクを貼る」方針とします。

## 2. ゴール / 非ゴール

### 2.1 ゴール

1. ログイン成功後の初期表示をカレンダー起点に統一する  
2. カレンダーからレポート一覧へ遷移できる導線を明示する  
3. 既存のレポート一覧機能（フィルタ・表示）は破壊しない

### 2.2 非ゴール

1. 認証方式（Firebase Auth）の変更  
2. カレンダーの週区切りロジック（UTC週）自体の変更  
3. レポート生成ロジック本体（内容生成）の変更

## 3. 変更方針

### 3.1 画面遷移（index/calendar）

- `public/index.html`:
  - 認証成功後、既定で `calendar.html` へ遷移する制御を追加する
  - レポート一覧を開くためのクエリパラメータ（例: `?view=reports`）を受け付ける

- `src/calendar_generator.py` / `public/calendar.html`:
  - レポート一覧への遷移リンク（`/` + query）をヘッダー付近に追加する
### 3.2 UI文言（案）

- カレンダー上部:
  - `📋 レポート一覧を開く`

## 4. 変更対象ファイル（予定）

1. `public/index.html`  
2. `src/calendar_generator.py`  
3. `public/calendar.html`（generator 再生成結果）  
4. `docs/03_components/login.md`  
5. `docs/03_components/calendar.md`

## 5. 検証計画

### 5.1 単体/静的確認

1. `python -m unittest discover tests`（既存回帰確認）
2. `ruff check .`（必要に応じて）

### 5.2 動作確認（ローカル）

1. ログイン成功時に `calendar.html` が初期表示されること
2. カレンダーからレポート詳細へ遷移できること
3. カレンダーからレポート一覧へ遷移できること

### 5.3 実行・デプロイ確認

- UI導線変更のため、`/debug-run`（原則 `USE_MOCK_DATA=false`）で生成結果を確認
- 必要に応じて `/deploy` で本番反映後、URLで確認

## 6. リスクと対策

1. 既存ユーザーが「ログイン後すぐレポート一覧」を期待している  
  - 対策: カレンダーから一覧への導線を常設し、遷移コストを最小化する
2. `public/calendar.html` の手編集との差分競合  
  - 対策: `src/calendar_generator.py` を修正し、再生成結果を正として扱う

## 7. 決定事項（ユーザー合意済み）

1. カレンダーにはレポート一覧リンクのみを設置し、デバッグ専用導線は実装しない
2. ログイン後の既定遷移はカレンダー固定とする
3. レポート一覧は `/?view=reports` で明示的に開く
