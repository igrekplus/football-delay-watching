## 要件の背景説明（ポイント）

### 1) YouTube Data API と映像の扱い

* *YouTube Data API が利用可能で、映像の検索・取得が可能なため、*
  過去対戦のハイライトや戦術分析動画を観戦前の文脈情報として配信することができる。
  **公式 API なので安定性が高い。** （YouTube Data API v3 使用）([api-sports][2])

---

### 2) 統計系データの活用

* Sportmonks などの API には **統計詳細（パス・シュート・xG 等）が含まれる**ため、
  スコアを表示せず *傾向や特性* を数字で比較する機能を作ることで、
  観戦前の理解を深める具体的な要素になる。([Sportmonks][3])

---

### 3) オッズ情報の扱い

* API-Football 等には **プリマッチオッズが含まれる**ため、
  これは *勝敗予測ではなく“どこに注目点があるか”* の指標に使うという要件で処理することがデータ利用の工夫になる。([api-football][4])

---

### 4) 怪我・出場停止データの注意

* Reddit の議論では、一部 API では怪我情報や出場停止の精度にばらつきがあるという感想があるため、
  この要件は *精度とカバレッジの検証* を考慮する必要がある。([Reddit][1])

---

## 何ができるようになるか（要件として整理）

* **動画リンクを直接レポートに含められるようになる**（No.1, No.2, No.11, No.14）
* **戦術的・統計的傾向を数字として示せるようになる**（No.3, No.4, No.10, No.13）
* **オッズを戦術文脈として活用できるようになる**（No.5）
* **怪我/出場停止を深掘りしたレポートが作れるようになる**（No.6）
* **観戦前のチェックリスト化・重点ポイント化ができるようになる**（No.12）
* **AI 文生成の精度と構造を向上できるようになる**（No.15）
* **多言語対応情報の拡張が可能になる**（No.8）
* **映像/統計/戦術をミックスした体験レポートが生成可能になる**（総合）

---

## 設計面の要点（共通）

* **結果を表示しない制約（ネタバレ禁止）** をすべての要件で前提とする
* **映像の引用は API による公式メタデータへのリンク形式** に限定
* **統計数値は直接スコアではなく傾向/比較指標として表現**
